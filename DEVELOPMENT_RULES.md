
# 移動支援アプリ 開発ルール

## プロジェクト概要
- **アプリ名**: 旅行サポートアプリ (travel-support-app)
- **対象ユーザー**: 障害者・高齢者・外国人観光客・一般ユーザー
- **開発体制**: 個人開発

## 技術スタック
- **フロントエンド**: React (Web) / JavaScript
- **パッケージ管理**: Node.js / npm
- **バージョン管理**: Git / GitHub
- **開発方針**: WebでデザインしてからモバイルApp化

## 必須機能
- ルート検索・ナビゲーション
- バリアフリー情報表示
- 多言語対応
- 音声読み上げ（アクセシビリティ）
- リアルタイム情報更新

## コーディングルール
- **コンポーネント**: 関数コンポーネント + Hooks
- **命名規則**: コンポーネントはPascalCase / 変数・関数はcamelCase
- **フォルダ構成**:
  - src/components … 再利用できる部品
  - src/pages … 各画面
  - src/hooks … カスタムHooks
  - src/utils … 便利な関数
  - src/locales … 多言語対応ファイル
  - src/assets … 画像・アイコン

## Git運用ルール
- **ブランチ戦略**: GitHub Flow
  - main … 常に動く状態を保つ本番ブランチ
  - feature/機能名 … 新機能の開発ブランチ
  - fix/バグ名 … バグ修正ブランチ
- **コミットメッセージ形式**:
  - feat: 新機能追加
  - fix: バグ修正
  - docs: ドキュメント更新
  - style: デザイン調整
  - refactor: コード整理
- **運用ルール**:
  - mainに直接コミットしない
  - 作業終了時は必ずgit pushする
  - 作業前は必ずgit pullする

## バージョン管理ルール
- バージョン管理を必ず行う
- バージョンごとに修正点・追加内容の記録を残す
- バージョン履歴は CHANGELOG.md に記載する

## 品質チェックルール
アプリ・コードの更新時は必ず以下を確認すること：
- 文字チェック（誤字・脱字）
- 文字のゆらぎチェック（表記の統一）
- シンタックスエラーチェック
- セキュリティ関連のプログラムは特に入念にチェックし、ミスを絶対に出さない

## 変更管理ルール
- 大幅な変更の際も、過去に指示した内容を必ず引き継いで実行すること
- ルールは今後追加される可能性があり、追加時はこのファイルを更新すること

## アクセシビリティ方針
- WCAG 2.1 AA準拠を目標
- 最小文字サイズ: 16px
- コントラスト比: 4.5:1以上
- aria-label / roleを適切に設定
