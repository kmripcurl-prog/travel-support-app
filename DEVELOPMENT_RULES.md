
# 移動支援アプリ 開発ルール

## プロジェクト概要
- **アプリ名**: 旅行サポートアプリ (travel-support-app)
- **対象ユーザー**: 障害者・高齢者・外国人観光客・一般ユーザー
- **開発体制**: 個人開発

## 技術スタック
- **フロントエンド**: React (Web) / JavaScript
- **パッケージ管理**: Node.js / npm
- **バージョン管理**: Git / GitHub
- **開発方針**: WebでデザインしてからモバイルApp化

## 必須機能
- ルート検索・ナビゲーション
- バリアフリー情報表示
- 多言語対応
- 音声読み上げ（アクセシビリティ）
- リアルタイム情報更新

## コーディングルール
- **コンポーネント**: 関数コンポーネント + Hooks
- **命名規則**: コンポーネントはPascalCase / 変数・関数はcamelCase
- **フォルダ構成**:
  - src/components … 再利用できる部品
  - src/pages … 各画面
  - src/hooks … カスタムHooks
  - src/utils … 便利な関数
  - src/locales … 多言語対応ファイル
  - src/assets … 画像・アイコン

## Git運用ルール
- **ブランチ戦略**: GitHub Flow
  - main … 常に動く状態を保つ本番ブランチ
  - feature/機能名 … 新機能の開発ブランチ
  - fix/バグ名 … バグ修正ブランチ
- **コミットメッセージ形式**:
  - feat: 新機能追加
  - fix: バグ修正
  - docs: ドキュメント更新
  - style: デザイン調整
  - refactor: コード整理
- **運用ルール**:
  - mainに直接コミットしない
  - 作業終了時は必ずgit pushする
  - 作業前は必ずgit pullする

## バージョン管理ルール
- バージョン管理を必ず行う
- バージョンごとに修正点・追加内容の記録を残す
- バージョン履歴は CHANGELOG.md に記載する

## 品質チェックルール
アプリ・コードの更新時は必ず以下を確認すること：
- 文字チェック（誤字・脱字）
- 文字のゆらぎチェック（表記の統一）
- シンタックスエラーチェック
- セキュリティ関連のプログラムは特に入念にチェックし、ミスを絶対に出さない

## 変更管理ルール
- 大幅な変更の際も、過去に指示した内容を必ず引き継いで実行すること
- ルールは今後追加される可能性があり、追加時はこのファイルを更新すること

## アクセシビリティ方針
- WCAG 2.1 AA準拠を目標
- 最小文字サイズ: 16px
- コントラスト比: 4.5:1以上
- aria-label / roleを適切に設定
## アプリコンセプト
「どんな人でも、どこへでも、確実に連れて行くAIコンシェルジュ」

### 対象ユーザー
- 障害者・高齢者・外国人観光客・一般ユーザー

### 対応言語
- 日本語・英語・中国語・韓国語・その他（ポルトガル語・スペイン語・ベトナム語・タガログ語等）

### 収益モデル
- フリーミアム（基本無料・プレミアム機能は有料）

### 対象エリア
- まず日本国内から開始・将来海外展開

## 主要機能一覧

### 基本移動サポート
- 現在地取得からスタート
- 行き先・出発日時をシンプルに確認
- 移動ルートの詳細提示
- 交通機関・飛行機・レンタカー・ホテル・レストラン・観光予約

### リアルタイム情報連携
- 気象情報（雨・雪・台風・暴風警報）
- 交通事故・渋滞情報
- 列車事故・遅延・運休情報
- 路面状況（凍結・浸水・工事）
- 催事・イベントによる混雑情報

### 危険情報表示
- 緊急アラートは画面全体・大きく・音声で表示
- 重要度別表示（最高・高・中・低）
- 経路上の危険情報（段差・暗い道・工事等）
- 災害情報による回避案内

### 移動追跡・緊急サポート
- 移動経路の逐次追跡
- 乗り換えミス検知・即時代替ルート提案
- 事前アラート（次の乗り換えまで3分等）
- 緊急時サポート（体調不良・迷子・紛失）
- 家族への緊急位置情報通知

### フレキシブルルート変更
- 移動中のワンタップ行き先変更
- 急な立ち寄り対応
- 立ち寄りによる到着時間への影響を即計算
- AI能動的提案（近くのおすすめスポット等）

### 徒歩安全サポート
- 夜間・危険ルート回避
- バリアフリールート優先
- 天候による歩行危険情報
- ユーザー別ルート最適化
  - 高齢者：平坦・休憩スポットあり
  - 車椅子：段差なし・スロープあり
  - 子連れ：安全な横断歩道・公園近く
  - 視覚障害者：音響信号機・点字ブロックあり

### 体調配慮機能
- 出発前の体調確認
- 移動距離・時間に応じた休憩提案
- 気温・天候による熱中症・寒さ対策アドバイス
- 緊急時の最寄り病院案内
- さりげなく・やかましくしない自然なサポート

### エンターテインメント・パーソナライズ
- 移動時間に合わせた情報提供（歴史・観光・豆知識）
- 待ち時間中のコンテンツ提案
- ユーザーの好み・履歴を記憶・学習
- 使うほど最適化されるパーソナライズ
- まるで長年の旅仲間のような関係性を構築

## AIコンシェルジュの基本姿勢
- さりげないサポート：必要な時に必要な情報だけ提供
- 通知の最小化：重要度の低い情報はまとめて表示
- 状況を察知：ユーザーが困っているサインを自動検知
- 学習・適応：使うほどそのユーザーに最適化
- 聞かれる前に必要な情報を提供する気の利くガイド
## パフォーマンス方針
- アプリの動作速度を最優先に考慮する
- 画面の表示速度を常に最適化する
- 不要な処理・通信を最小限にする
- データの読み込みは必要な分だけ・必要な時だけ行う
- リアルタイム処理でも動作が重くならない設計を徹底する
- 定期的にパフォーマンステストを実施する
- 高齢者・障害者が使う場面でも快適に動作することを保証する
